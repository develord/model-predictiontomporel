[T12] ⏳ Ensemble Model (NOUVEAU)
─────────────────────────────────────────────────────────────────────────
  Priorité:        P2 - HAUTE
  Catégorie:       Model
  Effort:          2 jours
  Dépendances:     T10, T11

  Description:
    - Ensemble: XGBoost + LightGBM + CatBoost
    - Voting pondéré par performance de chaque model
    - Apply to both classification AND regression

    Architecture:
    1. Train 3 models per task:
       - XGBoost (gradient boosting)
       - LightGBM (leaf-wise growth, faster)
       - CatBoost (handles categoricals well)

    2. Weighted voting:
       weights = [
           xgb_val_score / total_score,
           lgb_val_score / total_score,
           cat_val_score / total_score
       ]

       final_pred = sum(w * pred for w, pred in zip(weights, predictions))

    3. Confidence aggregation:
       # For classification
       final_confidence = weighted_average(model_confidences)

       # Only trade if majority agree on direction
       if not (xgb_pred == lgb_pred == cat_pred):
           confidence *= 0.7  # Penalize disagreement

  Rationale ChatGPT:
    "Ensemble reduces overfitting and variance. Different models
     capture different patterns. Voting improves robustness."

[T14] ⏳ TimeSeriesSplit Validation (NOUVEAU)
─────────────────────────────────────────────────────────────────────────
  Priorité:        P1 - CRITIQUE
  Catégorie:       Validation
  Effort:          1 jour
  Dépendances:     T11

  Description:
    - Replace simple train/test split with TimeSeriesSplit
    - Walk-forward validation: train on past, test on future
    - K-fold temporal split (avoid data leakage)

    Implementation:
    from sklearn.model_selection import TimeSeriesSplit

    tscv = TimeSeriesSplit(n_splits=5)
    scores = []

    for train_idx, val_idx in tscv.split(X):
        X_train, X_val = X[train_idx], X[val_idx]
        y_train, y_val = y[train_idx], y[val_idx]

        model.fit(X_train, y_train)
        score = evaluate(model, X_val, y_val)
        scores.append(score)

    # Mean score across folds
    cv_score = np.mean(scores)

  Rationale:
    - Time series data: future depends on past
    - Random CV leaks future info into training
    - TimeSeriesSplit respects temporal order

  Impact Attendu:
    + More realistic validation scores
    + Detect overfitting earlier
    + Better generalization to unseen data

  Métriques de Succès:
    ✓ TimeSeriesSplit implemented correctly
    ✓ CV scores stable across folds (not huge variance)

[T19] ⏳ Dynamic TP/SL Based on ATR (NOUVEAU)
─────────────────────────────────────────────────────────────────────────
  Priorité:        P3 - MOYENNE
  Catégorie:       Backtest
  Effort:          1 jour
  Dépendances:     T3, T16

  Description:
    - Adjust TP/SL based on current volatility (ATR)
    - Low volatility → tighter TP/SL
    - High volatility → wider TP/SL

    Formula:
    ATR_percentile = percentile(ATR, lookback=60 days)

    if ATR_percentile < 0.3:  # Low volatility
        tp_mult = 0.7
        sl_mult = 0.8
    elif ATR_percentile > 0.7:  # High volatility
        tp_mult = 1.3
        sl_mult = 1.2
    else:  # Normal
        tp_mult = 1.0
        sl_mult = 1.0

    # Apply to regression target or default TP/SL
    final_tp = base_tp * tp_mult
    final_sl = base_sl * sl_mult

  Rationale:
    - Fixed TP/SL doesn't adapt to market conditions
    - Low vol: take profits quickly (limited upside)
    - High vol: give room to breathe (avoid premature stops)

  Impact Attendu:
    + Adaptive risk management
    + Better trade exits
    + Estimated +3-7% ROI improvement

  Métriques de Succès:
